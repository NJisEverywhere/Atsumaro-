# 中間地点モード 仕様書

## 概要

全メンバーの出発地から、移動時間が均等になる中間地点（駅）を算出・提案するモード。

---

## 計算ロジック

### アルゴリズム

```
① 全メンバーの出発地（緯度経度）を取得
      ↓
② 出発地の地理的重心（平均緯度・平均経度）を算出
      ↓
③ 重心付近の駅を Google Places API で検索
   （type: transit_station, 半径 3〜5km 程度）
      ↓
④ Distance Matrix API で「全メンバー → 各候補駅」の移動時間を一括取得
      ↓
⑤ 各候補駅ごとに「最大移動時間 - 最小移動時間（差分）」を計算
      ↓
⑥ 差分が小さい順にランキング → 上位3〜5駅を提案
```

### 均等さの定義

MVP では **移動時間の差分が最小**になる駅を優先する。

### 将来対応：早・楽・安の切り替え（Yahoo乗換案内ライク）

| モード | ロジック |
|---|---|
| **早い** | 全員の移動時間の合計が最小 |
| **楽** | 移動時間の差分（不公平感）が最小 ← MVP |
| **安い** | 運賃の合計が最小（要: 運賃取得API） |

---

## 対応人数

2〜5人

---

## 出発地の入力方法

Yahoo乗換案内ライクな UI を参考に、以下の2つを併用する。

- **テキスト検索**：駅名・住所を文字で入力（Google Places Autocomplete を使用）
- **GPS自動取得**：ボタン一つで現在地をセット（`navigator.geolocation`）

---

## 画面遷移

```
ホーム
 └─ 中間地点モード選択
      └─ ① 出発地入力画面（人数分）
           └─ ② 計算中画面（ローディング）
                └─ ③ 結果一覧画面（候補駅リスト）
                     └─ ④ 候補詳細画面
```

---

## 各画面の仕様

### ① 出発地入力画面

- 人数選択（2〜5人）
- 各メンバーの出発地入力欄
  - テキスト入力 + Places Autocomplete
  - GPS取得ボタン
- 「探す」ボタン（全員分入力済みで活性化）

### ② 計算中画面

- ローディングアニメーション
- 「X人分のルートを計算中...」メッセージ

### ③ 結果一覧画面

- 候補駅カード（3〜5件）
  - 駅名・路線名
  - 全員の移動時間一覧（例：「A: 20分 / B: 22分 / C: 18分」）
  - 公平スコア表示（移動時間の最大差）
- 地図上にピン表示
- スポットカテゴリフィルター（カフェ / 居酒屋 / ご飯 / ショッピング）

### ④ 候補詳細画面

- 選んだ駅の周辺スポット一覧（Google Places API）
- 各メンバーのルート個別表示
- グループ共有ボタン（リンクコピー）

---

## 使用API

| API | 用途 |
|---|---|
| Google Places Autocomplete | 出発地のテキスト入力補完 |
| Google Places API（nearby search） | 重心付近の候補駅検索 |
| Google Distance Matrix API | 全メンバー → 各候補駅の移動時間取得 |
| Google Places API（place details） | 周辺スポット検索 |
